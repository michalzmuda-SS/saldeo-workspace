# PROPOSALS — kandydaci na nowe petle / skille

> System proponuje, Michal akceptuje. Zmien `status: proposed` -> `approved`, zeby Claude to zbudowal
> (albo napisz "zaakceptuj <id>"). Odrzucenie: `status: rejected` + powod.

Format wpisu (najnowsze na gorze):

```
### <id> — <tytul>
- status: proposed | approved | rejected
- data: RRRR-MM-DD
- typ: skill | loop
- model/agent: <ssa-product|par-analytics|dt-data / agent>
- trigger: <kiedy sie uruchamia>
- kroki: <skrot workflow>
- BRAMKA: <weryfikowalne nie>   # bez bramki propozycja petli jest odrzucana
- stop-rules: <max prob / sensitive stop>
- wartosc / koszt: <po co, oszacowanie>
```
